# 실제 동작 테스트 가이드

## 0. 사전 준비
- manifest.json의 oauth2.client_id를 실제 값으로 설정
- Chrome 확장프로그램 관리 페이지( chrome://extensions/ )에서 개발자 모드 ON
- "압축해제된 확장프로그램 로드"로 D:\git\google-calendar-extension 선택

## 1. 기본 플로우 테스트(로그인 상태)
1. Google 계정으로 Chrome 로그인 상태 확인
2. 임의 웹페이지에서 "2026년 2월 10일" 텍스트 드래그 선택
3. 입력 UI 표시 확인 (날짜 자동 채움)
4. 제목/시간/메모 입력 후 저장
5. Google Calendar에서 이벤트 생성 여부 확인

## 2. 비로그인 플로우 테스트
1. Chrome에서 Google 로그아웃 또는 토큰 제거
2. 동일하게 날짜 드래그 → 저장 클릭
3. OAuth 로그인 UI 표시 확인
4. 로그인 완료 후 자동 저장 확인

## 3. 파싱 실패 케이스
1. 날짜 형식이 아닌 텍스트 선택 (예: "내일 오후")
2. "Date parse failed" 메시지 표시 확인
3. 날짜 수동 입력 후 저장 가능 확인

## 4. mm/dd 파싱 테스트
1. Options에서 mm/dd 파싱 활성화 확인
2. 텍스트 "2/3" 선택
3. 현재 연도 적용된 날짜 자동 채움 확인

## 5. 제목 필수 검증
1. 날짜만 입력하고 제목 비워둔 채 저장 시도
2. "Title is required" 경고 확인

## 6. 네트워크 오류 처리
1. 네트워크 차단 상태에서 저장 시도
2. 실패 메시지 표시 확인

## 7. 토큰 만료 처리
1. OAuth 토큰 제거(로그아웃) 후 저장 시도
2. 재로그인 흐름으로 전환되는지 확인

## 8. 옵션 저장 확인
1. Options에서 기본 시작 시간/소요시간 변경
2. 저장 후 다시 열어 값 유지 확인
3. 저장 시 기본 시간 적용 확인
